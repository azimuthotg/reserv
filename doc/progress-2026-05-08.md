### 6. IoT Toggle ดีดกลับ

**ปัญหาเดิม:** กดปุ่ม toggle UI เปลี่ยน ดีดกลับ เปลี่ยนอีกที (flicker 3 ครั้ง)

**สาเหตุ:** Race condition — Django เรียก `_ha_get_state()` ทันทีหลัง toggle ก่อน HA เปลี่ยนสถานะจริง และ poll 2 วิยิงทับ

**สิ่งที่แก้:**
- Frontend: เชื่อ optimistic state ไม่ override ด้วย API response หลัง toggle สำเร็จ
- Frontend: หน่วง `isToggling` 3 วินาทีหลัง toggle ให้ HA ตั้งตัวก่อน poll sync
- Backend: `time.sleep(0.6)` ก่อน `_ha_get_state()` เป็น safety net

**ไฟล์:** `booking/views.py`, `booking/templates/booking/room_control.html`
