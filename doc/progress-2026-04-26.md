### 1. ระบบปิดห้องชั่วคราว (RoomClosure)

**ปัญหา:** เดิมสามารถปิดบริการได้เฉพาะ "วันหยุด" ซึ่งปิดทุกห้องพร้อมกัน ไม่สามารถปิดเฉพาะห้องหรือเฉพาะช่วงเวลา AM/PM ได้

**วิธีแก้:** สร้าง Model ใหม่ `RoomClosure`

| Field | ประเภท | คำอธิบาย |
|---|---|---|
| `room` | ForeignKey → Room | ห้องที่ปิด |
| `date` | DateField | วันที่ปิด |
| `period` | CharField | am / pm / all_day |
| `reason` | CharField | สาเหตุ |
| `is_active` | BooleanField | เปิด/ปิดใช้งาน |

**Migration:** `0010_roomclosure`

**หน้าจัดการ UI ใน `/manage/`:**
- `/manage/closures/` — list กรองตามปีและห้อง
- `/manage/closures/add/` — เพิ่มรายการ
- `/manage/closures/<id>/edit/` — แก้ไข
- `/manage/closures/<id>/toggle/` — เปิด/ปิดใช้งาน
- `/manage/closures/<id>/delete/` — ลบ

**API:** `GET /api/room-closure/?room=<key>&date=<YYYY-MM-DD>`
- คืน list closure ของห้อง+วัน สำหรับ booking.html ใช้เช็คทันที

### 2. แสดงวันหยุดในปฏิทิน FullCalendar

**ปัญหา:** ปฏิทินแสดงเฉพาะการจอง ไม่แสดงวันหยุด ผู้ใช้ไม่ทราบว่าวันใดปิดบริการ

**วิธีแก้:** แก้ `calendar_events` API ให้คืน `HolidayDate` เป็น event ด้วย

```
event สีแดง + allDay = true
title: "ปิดบริการ — {ชื่อวันหยุด}"
```

**เพิ่ม:** legend วันหยุด (จุดสีแดง) ใน calendar + popup แสดงรายละเอียดเมื่อกด

---

### 3. แจ้งเตือนทันทีเมื่อเลือกวันปิดบริการในหน้าจอง

**ปัญหา:** ผู้ใช้เลือกวันหยุดหรือวันที่ห้องปิด แล้วกรอกฟอร์มจนเสร็จ กด submit ถึงจะทราบว่าจองไม่ได้ เสียเวลา

**วิธีแก้:** แก้ `booking.html` ให้ตรวจสอบทันทีหลังเลือกวันที่

- วันหยุด → แสดงชื่อวันหยุดพร้อม error ทันที (ไม่ล้างฟอร์ม)
- RoomClosure → เรียก API ตรวจสอบ → แสดง warning ทันที

**ปรับ:** `holidays_json` จาก `Set` เป็น `Dict { date: description }` เพื่อแสดงชื่อวันหยุดในข้อความแจ้งเตือน

### 5. IoT Monitor — ระบบตรวจสอบอุปกรณ์

**ปัญหา:** ไม่มีระบบตรวจสอบสถานะ IoT รวมศูนย์ ผู้ดูแลไม่ทราบเมื่ออุปกรณ์ offline ผู้ใช้เข้าห้องแล้วใช้งานไม่ได้

**วิธีแก้:** สร้างระบบ Monitor แบบ Morning Check + On-demand

**หน้า `/manage/iot-monitor/`:**
- โหลดสถานะทุกอุปกรณ์ทุกห้องอัตโนมัติเมื่อเปิดหน้า
- badge ✅ ปกติ / ⚠️ ผิดปกติ ต่อห้อง
- ปุ่ม **Refresh** — ดึงสถานะ real-time แบบ on-demand
- ปุ่ม **แจ้งกลุ่ม LINE** — ส่งสรุปสถานะไปกลุ่มผู้ดูแล (manual)

**รายงานเช้าอัตโนมัติ (`morning_iot_report`):**
- ส่ง 1:1 ให้ผู้รับผิดชอบ (สุรนาถ สุพรรณ) ผ่าน `IOT_ADMIN_LINE_ID`
- ตั้ง Windows Task Scheduler รันทุกวัน **07:30 น.**
- ไม่ส่งกลุ่ม — ลด noise สำหรับสมาชิกกลุ่มคนอื่น

**ตัวอย่างข้อความรายงาน:**
