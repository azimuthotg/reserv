### 1. IoT Monitor — เพิ่ม Device Control (เปิด/ปิดอุปกรณ์)

**ปัญหา:** หน้า `/manage/iot-monitor/` ดูสถานะได้แต่ยังควบคุมอุปกรณ์จาก UI ไม่ได้

**วิธีแก้:** เพิ่ม endpoint `manage_iot_device_control` และ UI toggle switch

**Backend:**
- เพิ่ม `_ha_call_service(entity_id, action)` เรียก Home Assistant REST API
  - `POST /api/services/{domain}/turn_on|turn_off`
  - detect domain อัตโนมัติจาก `entity_id` prefix (เช่น `switch.xxx` → domain = `switch`)
- เพิ่ม `manage_iot_device_control(request, pk)` view รับ POST `action=on|off`
  - sleep 0.8 วิหลัง call HA แล้ว re-fetch สถานะใหม่
  - คืน `JsonResponse({ success, state, error })`
- เปิดให้ staff ทุกคนใช้ได้ (ไม่จำกัดแค่ superuser)

**URL:** `manage/iot-monitor/control/<int:pk>/` ชื่อ `manage_iot_device_control`

### 3. IoT Monitor — Auto-poll ประหยัด Resource (Option C)

**พฤติกรรม:**
- Poll ทุก **10 วินาที** เมื่อ browser tab active

### 4. Custom Template Tag `th_filters` — วันที่/เวลาเป็น พ.ศ.

**ปัญหา:** วันที่ทั้งระบบแสดงเป็น ค.ศ. และไม่มี "น." ต่อท้ายเวลา

**วิธีแก้:** สร้าง custom template tag ใน `booking/templatetags/th_filters.py`

| Filter | ผลลัพธ์ตัวอย่าง |
|---|---|
| `th_date` | `28/04/2569` |
| `th_date_long` | `28 เมษายน 2569` |
| `th_weekday` | `จันทร์` |
| `th_datetime` | `28/04/2569 10:30 น.` |
| `th_datetime_sec` | `28/04/2569 10:30:45 น.` |

**แก้ 10 templates** ให้ `{% load th_filters %}` และใช้ filter ใหม่:
- `success.html`, `bookings.html`, `booking_logs.html`, `dashboard.html`
- `holidays.html`, `closures.html`, `line_users.html`, `line_user_detail.html`
- `staff.html`, `rooms.html`

### 6. หน้า Daily Schedule `/manage/daily/`

**ฟีเจอร์:** แสดงตารางการจองแบบ Gantt Timeline + รายการ cards รายวัน

**Zone 1 — Date Navigation:**
- ปุ่มย้อน/ถัดไป, แสดงวันที่เป็น พ.ศ.
- นาฬิกา live (วันนี้เท่านั้น)
- สถิติ "ใช้งานอยู่" + "จองวันนี้"

**Zone 2 — Gantt Timeline:**
- ช่วงเวลา **07:00–17:00 น.** (TLINE_START=420, TLINE_RANGE=600)
- แต่ละห้องเป็น 1 แถว แสดงช่วงเวลาเปิด-ปิดและ booking blocks
- เส้น now-line สีแดง (เฉพาะวันนี้) อัปเดตทุก 30 วิ
- Booking blocks ระบายสี: เขียว (active) / น้ำเงิน (upcoming) / เทา (done)

**Zone 3 — Booking Cards:**
- เรียงตามเวลา
- Badge สถานะ real-time: กำลังใช้งาน / รอการใช้งาน / เสร็จแล้ว
- แสดง ชื่อกลุ่ม, ชื่อผู้จอง, คณะ, สถานะ check-in

**Alert วันหยุด/ปิดห้อง:**
- เหลือง: วันหยุดราชการ (แสดงชื่อวันหยุด)
- น้ำเงิน: ห้องที่ปิดชั่วคราว + ช่วงเวลา

**Backend:** `manage_daily_schedule()` ใน `manage_views.py`
- pre-calculate `left_pct`, `width_pct`, `open_left_pct`, `open_width_pct` ส่งให้ template
- `bookings_json` serialize สำหรับ JS real-time update
- query `holiday` + `closures` ของวันที่เลือก
