## การตัดสินใจ: ไม่สร้างเป็น Room

flip gate ไม่สังกัดห้องจองใด และมีตารางเวลาของตัวเอง (07:20-17:00) ต่างจากห้องจอง (ปิด 16:30)
ถ้าสร้างเป็น `Room` จะลาก `open_time`/`close_time` ที่ไม่ตรงความจริงติดมาด้วย (เวลาใน Room
มีความหมายกับ "การจอง" เท่านั้น ซึ่ง gate ไม่เกี่ยว) และไปโผล่ในหน้าจอง ปฏิทิน สถิติโดยไม่จำเป็น

เลือกทำเป็น **"อุปกรณ์ส่วนกลาง"** แทน = `RoomDevice` ที่ `room=None` + `group_name`
ฝั่ง Django **ไม่มี logic เรื่องเวลาของ gate เลย** ปล่อยให้ HA automation คุมตามเดิม
หน้า monitor ทำหน้าที่แค่ดูสถานะ + สั่ง override ด้วยมือ

### หมายเหตุการออกแบบ

- **รวม logic ไว้ที่ `_iot_cards()` แหล่งเดียว** — เดิม 4 จุด (หน้า monitor, refresh, แจ้งกลุ่ม LINE,
  morning report) ต่างคนต่าง `for room in rooms: room.devices.all()` เอง ถ้าไม่ยุบต้องแก้ 4 ที่
- **key ของการ์ดใช้ md5 ย่อ 8 หลัก** ไม่ใช่ `slugify` เพราะ `slugify(allow_unicode=True)` **ตัดสระและ
  วรรณยุกต์ไทยทิ้ง** (`อุปกรณ์ส่วนกลาง` → `อปกรณสวนกลาง`) ทำให้ชื่อกลุ่มที่ต่างกันเหลือ slug เดียวกันได้
  key นี้ใช้เป็น DOM id เท่านั้น ไม่ต้องอ่านรู้เรื่อง
- **JSON ของ `/manage/iot-monitor/refresh/` เปลี่ยน shape** จาก `{rooms:[{room_name, room_key,…}]}`
  เป็น `{cards:[{name, key,…}]}` — ถ้ามี consumer อื่นนอกจากหน้านี้ต้องแก้ตาม (ตรวจแล้วไม่มี)
- **เพิ่ม/ลบอุปกรณ์กลุ่มผ่าน Django Admin** เพราะ `manage_room_devices` เป็นหน้าราย "ห้อง"
  อุปกรณ์กลุ่มไม่มีห้องให้เข้าหน้านั้น
