# ส่วนที่ 3 — แก้ metric No-show ให้มีความหมายจริง (หลังทดสอบ prod รอบแรก)

## ที่มา

ทดสอบหน้า analytics บน prod จริง (Monitor แคปหน้าจอมาให้) — หน้า render ครบถูกต้อง แต่พบว่า
**No-show = 0.0% (0/125)** ซึ่งสวยเกินจริง

**สาเหตุ:** ระบบมี job `send_reminders.py` ([send_reminders.py:143](../booking/management/commands/send_reminders.py))
ที่ **auto-cancel booking ที่ไม่ check-in ภายใน 15 นาทีหลังเริ่ม** → เปลี่ยน `status='cancelled'` +
บันทึก `BookingLog action='auto_cancelled'` ดังนั้น no-show จริงจะถูกย้ายไปเป็น cancelled หมด
metric เดิมที่นับจาก "confirmed + เลยเวลา + ไม่ check-in" เลยได้ ~0 เสมอ ส่วน no-show จริงไปซ่อน
ปนอยู่ในอัตรายกเลิก (35.5% / 71)

## สิ่งที่แก้ (`manage_analytics()` + `analytics.html`)

- **No-show** เปลี่ยนมานับจาก `BookingLog action='auto_cancelled'` ในช่วง (นับ distinct booking)
  - No-show rate = no-show ÷ (booking ที่ผ่านเวลาแล้วยัง confirmed + no-show)
- **แยกอัตรายกเลิกเป็น 2 ก้อน:**
  - No-show (auto-cancel) = ไม่มา/ไม่ check-in → การ์ด No-show
  - **ยกเลิกโดยผู้ใช้** = cancelled ทั้งหมด − no-show → การ์ดใหม่ "ยกเลิกโดยผู้ใช้" (แทน "อัตรายกเลิก" เดิม)
- คอลัมน์ No-show ต่อผู้ใช้ในตารางจองถี่ นับจาก auto_cancelled log เช่นกัน (เดิมได้ 0 หมด)
- ตารางยกเลิกกระชั้นชิด: เพิ่ม guard ข้าม booking ที่ reason เป็น auto-cancel (กันปน no-show)

## วิธีตรวจสอบ

- **Automated test** — อัปเดต `ManageAnalyticsTests` ให้ seed no-show แบบ auto-cancel (+BookingLog)
  แล้ว assert `no_show_count`/`no_show_denom`/`user_cancelled` ถูกต้อง → **7/7 ผ่าน**
- **เบราว์เซอร์จริง** (preview + seed auto-cancel 14 รายการ) — ยืนยัน No-show ขึ้น 19.2% (10/52)
  ไม่ใช่ 0% แล้ว, การ์ด "ยกเลิกโดยผู้ใช้" แยกออกมา (1.9%), คอลัมน์ No-show ต่อผู้ใช้มีเลขจริง (3/2/1/1),
  ตารางยกเลิกกระชั้นชิดไม่ปน auto-cancel, กราฟ/ไม่มี console error

**สถานะ:** ✅ แก้ + ตรวจผ่าน (test + browser) + **ทดสอบ prod ผ่านแล้ว 2026-07-09**
- No-show ขึ้น **23.5% (39/166)** (เดิม 0%) — มีความหมายจริงแล้ว
- ยกเลิกโดยผู้ใช้ **16.0% (32)** แยกจาก no-show
- ตรวจความถูกต้อง: 39 (no-show) + 32 (ยกเลิกผู้ใช้) = 71 = ยอด cancelled เดิมพอดี ✓
- คอลัมน์ No-show ต่อผู้ใช้ขึ้นเลขจริง (5/2/4/1...) ไม่ใช่ 0 หมด

**→ หน้า analytics `/manage/analytics/` ปิดงานแล้ว (prod verified)**
