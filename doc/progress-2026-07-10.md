# Progress 2026-07-10 — สมาชิกถาวรไม่บังคับเลขบัตร (รองรับ VVIP เช่น นายกสภามหาวิทยาลัย)

**เครื่องมือ:** Claude Code edit (แก้ทั้ง reserv และ apiproject)

---

## ที่มา

ต้องการให้บุคคลสำคัญ (VVIP) เช่น นายกสภามหาวิทยาลัยนครพนม เข้าห้องสมุดด้วย QR ได้
โดย**ไม่ต้องให้เลขบัตรประชาชน** — ผู้ใช้เลือกแนวทาง**แก้น้อยสุด**: ไม่เพิ่ม endpoint ใหม่
ไม่แตะ model ไม่มี field ตำแหน่ง/badge ใช้ flow สมาชิกถาวรเดิม 100% แค่เว้นช่องเลขบัตรว่างได้

**หลักการ:** เมื่อไม่กรอกเลขบัตร api ออก **รหัสอ้างอิง `V` + เลขสุ่ม 12 หลัก** (รวม 13 ตัว
เท่า PK เดิม ไม่ชนเลขบัตรจริงซึ่งเป็นตัวเลขล้วน) — กลไก permanent_code/เช็คประตูเดิมใช้ได้ทันที

## สิ่งที่แก้

### ฝั่ง apiproject (ไฟล์เดียว: `apiapp/views_v2.py` + tests)
- เพิ่ม helper `_gen_external_ref_id()` — gen `V`+12 หลักสุ่ม กันชนด้วย exists() loop
  (แนวเดียวกับ `_gen_permanent_code()` เดิม)
- `permanent_register()`: citizen_id ว่าง → gen รหัสอ้างอิงให้; กรอกมา → ตรวจ checksum เหมือนเดิม
- url regex 5 จุด (detail/approve/revoke/delete/photo): `[0-9]{13}` → `[0-9V][0-9]{12}`
- `check_external` / model / serializer / migration — **ไม่แก้**
- tests ใหม่ `PermanentRegisterNoCitizenIdTests` 4 เคส (gen ref id, checksum ผิดยัง 400,
  flow เต็ม register→approve→เช็คประตู 200, url resolve รับ `V…`)

### ฝั่ง reserv
- `booking/manage_views.py` — `manage_external_register()`: กรณี 201 redirect ไป detail
  ด้วย citizen_id จาก **response JSON** (กรณีเว้นว่าง ID จริงถูก gen ฝั่ง api)
- `booking/templates/booking/manage/external_form.html` — ช่องเลขบัตรเอา `required` ออก
  + hint "เว้นว่างได้สำหรับบุคคลสำคัญ ระบบจะออกรหัสอ้างอิงให้อัตโนมัติ"
- `external_list.html` / `external_detail.html` — รายที่ ID ขึ้นต้น `V` แสดง "—"/"รหัสอ้างอิง"
  แทนเลขบัตร (เปลี่ยนเฉพาะการแสดงผล ปุ่มอนุมัติ/ระงับ/ลบ/รูปใช้ของเดิม)

  mock `_npu_v2_request`)
