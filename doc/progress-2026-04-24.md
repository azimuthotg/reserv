## สิ่งที่ดำเนินการในวันนี้

### 1. ระบบ Check-in และ Auto-cancel

เพิ่ม flow check-in เพื่อยืนยันการมาใช้ห้องจริง และ auto-cancel อัตโนมัติถ้าไม่ check-in

**Model เพิ่ม field ใน `Booking`:**

| Field | ประเภท | คำอธิบาย |
|---|---|---|
| `checked_in` | BooleanField | สถานะ check-in |
| `checked_in_at` | DateTimeField (null) | เวลาที่ check-in |

**Migration:** `0009_checkin_fields`

**API เพิ่ม:**
- `POST /api/checkin/` — รับ `{ userId, bookingId }` ตรวจ window -15/+15 นาทีของ start_time

**Check-in Window:**
```
start - 15 นาที  ← check-in เปิด (LINE notification ส่ง)
start_time       ← เริ่มเวลาจอง
start + 15 นาที  ← check-in ปิด / auto-cancel ถ้าไม่ check-in
```

**2 วิธี Check-in:**
1. กดปุ่ม "✅ Check-in" ในการ์ดจองของฉัน (landing page)
2. เปิดหน้า `/room-control/` → auto check-in ผ่าน LIFF

**Auto-cancel (`send_reminders.py`):**
- ทำงานทุก 1 นาทีผ่าน Task Scheduler
- ถ้า `now >= start + 15min` และ `checked_in = False` → cancel + แจ้ง LINE + `BookingLog(action='auto_cancelled')`

### 2. เปลี่ยนแจ้งเตือน 15 นาทีเป็น LINE Button Template

**จาก:** ข้อความธรรมดา + ลิงก์ `/room-control/` (เข้าไม่ได้ก่อนเวลา)

**เป็น:** Button Template พร้อมปุ่ม **"✅ Check-in เลย"** ลิงก์ไป `/reserv/` (landing page)

```
┌──────────────────────────────┐
│ ⏰ อีก 15 นาทีถึงเวลาใช้พื้นที่ │
│ 📍 ห้อง / 📅 วันที่ / 🕐 เวลา  │
│ กรุณา Check-in ภายใน XX:XX น. │
│ หากไม่ check-in = auto-cancel  │
├──────────────────────────────┤
│       [ ✅ Check-in เลย ]     │
└──────────────────────────────┘
```

เหตุผล: landing page รองรับทุก state — ก่อนเวลา (ปุ่ม Check-in) และระหว่างใช้ (ปุ่มควบคุมอุปกรณ์)

### 3. แก้ปุ่มควบคุมอุปกรณ์วิ่งผิดห้อง

**ปัญหา:** มี booking 2 ห้องพร้อมกัน กดปุ่มควบคุมห้องใดก็ไปที่ MINI THEATER เสมอ

**สาเหตุ:**
- ปุ่มใช้ `href="room-control/"` ตายตัว ไม่ส่ง room_key
- `room_control.html` ใช้ `.find()` เจอห้องแรกก็หยุด

**วิธีแก้:**
- `landing.html`: เปลี่ยนเป็น `href="room-control/?room=${b.room_key}"`
- `room_control.html`: อ่าน `?room=` จาก URL แล้วกรอง booking ตาม room_key

### 4. บล็อกการจองสล็อตที่ start_time ผ่านไปแล้ว

**ปัญหา:** `create_booking` เช็คแค่ `b_date < today` ทำให้จองสล็อตที่เริ่มแล้วในวันเดียวกันได้ เช่น เวลา 10:39 จอง slot 10:30 → เหลือเวลา check-in แค่ 6 นาทีก่อน auto-cancel

**วิธีแก้:** เพิ่มเงื่อนไข
```python
if b_date == date.today() and s_time <= timezone.localtime(timezone.now()).time():
    return error('ไม่สามารถจองสล็อตที่เริ่มแล้วหรือผ่านไปแล้ว')
```

### 5. ปรับปุ่มยกเลิกการจองตามสถานะ

**ปัญหา:** ปุ่มยกเลิกแสดงทุกการ์ดแม้หมดเวลาหรือ checked_in แล้ว

**Logic ใหม่:**

| สถานะ | การ์ด | Action |
|---|---|---|
| ยกเลิกแล้ว | dim + ขีดฆ่า | badge "ยกเลิกแล้ว" |
| หมดเวลาแล้ว | dim (.past) | badge "หมดเวลาแล้ว" |
| กำลังใช้งาน | ปกติ | ปุ่มควบคุม + badge "เช็คอินแล้ว" |
| check-in window | ปกติ | ปุ่ม Check-in + ปุ่มยกเลิก |
| ยังไม่ถึงเวลา | ปกติ | ปุ่มยกเลิก |

`canCancel = !isCancelled && !isPast && !isStarted && !checked_in`
