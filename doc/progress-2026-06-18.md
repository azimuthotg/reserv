## สิ่งที่ดำเนินการในวันนี้ (ฝั่ง reserv)

### 1) ส่ง LINE context ไปให้ API (commit 86e8b4e)
- `_verify_ldap(username, password, line_uid, display_name, user_type)` — แนบ
  `userId/displayName/user_type` ใน body ของ POST `auth_ldap` (ส่งทั้งเคสสำเร็จและจะ fail)
- `_register_npu_user(user_id, user_ldap, user_type, display_name)` — แนบ `displayName` เพิ่ม
- `register_page()` — ส่งตัวแปรที่มีอยู่แล้ว (`user_id, display_name, user_type`) เข้าไปครบ

### 2) ดึงชื่อ LINE จาก UID เป็น fallback (commit 7191135)
- เพิ่ม `_fetch_line_display_name(user_id)` — `GET https://api.line.me/v2/bot/profile/{userId}`
  ด้วย `settings.LINE_CHANNEL_ACCESS_TOKEN` (token เดิมที่ `_push_line` ใช้อยู่)
- `register_page()` — ถ้า `display_name` ว่าง (LIFF ไม่ส่งชื่อมา) จะดึงจาก LINE ก่อนส่งต่อ API
  → Monitor เห็นชื่อทุกแถวที่มี UID ทั้งเคสสำเร็จและไม่สำเร็จ
- เงื่อนไข: นักศึกษาต้อง add friend OA (ซึ่งใช่อยู่แล้ว เพราะ reserv ส่ง LINE หาเขาได้)
  ถ้าดึงไม่ได้ → คืน '' ไม่ทำให้ผูกบัญชีพัง
